## ⚡ Engineering Philosophy

> “Correctness first. Performance next. Abstractions last.”

- Invariants > assumptions  
- Measure before optimizing  
- Systems should fail **predictably**, not silently  

---

## 🧪 Engineering Signals

- Designed systems with **explicit invariants** eliminating **>90% invalid states**  
- Built concurrency-safe workflows under **real-world contention scenarios**  
- Reduced critical workflow latency from **~6s → 0.8s** via system-level optimizations  
- Debugged memory + threading issues using **gdb and profiling tools**  
- Focus on **predictable failure modes over silent corruption**

---

## 🏗️ System Design Thinking

- Prefer **bounded systems** (backpressure-aware design) over unbounded growth  
- Separate **control plane vs execution plane**  
- Design for **observability first** (logs, metrics, failure surfaces)  
- Enforce correctness via **invariants at system boundaries**
